# 像素风横版动作游戏设计

## 目标

制作一个可以在浏览器运行的像素风横版动作小游戏。玩家控制拿剑的小勇者，通过短关卡、击败小怪，并最终打败 Boss 获胜。

第一版重点是做出完整可玩的核心循环：移动、跳跃、平台碰撞、剑攻击、敌人、Boss、受伤、胜利、失败、键盘和手柄输入。

## 技术路线

使用原生 HTML、CSS 和 JavaScript 开发，主要画面通过 Canvas 绘制。

不使用 Phaser 等游戏框架。这样第一版更适合学习游戏基础逻辑，包括游戏循环、输入、重力、碰撞、攻击判定和实体状态。

## 第一版范围

- 一张横版关卡
- 一个玩家角色
- 一种普通小怪
- 一个 Boss
- 一套剑攻击
- 玩家血量、敌人血量、Boss 血量
- 玩家受伤、短暂无敌、死亡失败
- Boss 被击败后的胜利状态
- 键盘和手柄控制
- 程序绘制的像素风画面，不依赖外部图片素材

暂不包含多关卡、背包、升级、复杂技能、远程攻击、存档、音效和外部美术资源。

## 操作设计

键盘：

- `A / D` 或方向键：左右移动
- `W / Space`：跳跃
- `J`：挥剑攻击
- `R`：胜利或失败后重开

手柄：

- 左摇杆或十字键：左右移动
- 南键：跳跃，常见为 Xbox `A` 或 PlayStation `×`
- 西键或东键：攻击，常见为 Xbox `X/B` 或 PlayStation `□/○`
- `Start/Menu`：胜利或失败后重开

输入层会把键盘和手柄统一转换为动作状态：`left`、`right`、`jump`、`attack`、`restart`。游戏逻辑只读取动作状态，不直接关心输入设备。

## 玩法设计

玩家从关卡左侧出生，向右前进。中间区域包含平台和小怪，右侧是 Boss 区域。玩家需要跳过平台、用剑击败或绕过小怪，并在 Boss 区击败 Boss。

玩家有 5 点生命。普通小怪碰到玩家造成 1 点伤害。Boss 冲撞命中玩家造成 1 点伤害。玩家受伤后会进入短暂无敌时间，避免连续扣血。

玩家的剑攻击是角色面前的短矩形判定框。攻击持续时间很短，并带有冷却时间，避免连续无脑挥剑。

## 敌人设计

普通小怪：

- 外观是像素风小史莱姆或小兽
- 在固定平台范围内左右巡逻
- 血量为 2 点
- 接触玩家造成 1 点伤害
- 被剑击中后扣血、后退一点并短暂闪烁
- 血量归零后从关卡中消失

Boss：

- 外观比玩家大 2-3 倍
- 血量为 8 点
- 第一版只有一种攻击模式：靠近玩家时短距离冲撞
- 冲撞前有短暂停顿，给玩家反应时间
- 被剑击中后扣血，并进入短暂无敌时间
- Boss 血量归零后进入胜利状态

## 关卡设计

关卡为一张短横版地图：

- 左侧是玩家出生点
- 中间有 2-3 个平台，用于练习跳跃
- 中间区域放置 1-2 个普通小怪
- 右侧是 Boss 区域
- 顶部 UI 显示玩家血量和 Boss 血量

第一版采用固定的接近单屏布局，不做摄像机跟随。这样玩家、平台、小怪和 Boss 区都能在一张画面中被理解和调试，后续版本再扩展为横向卷轴地图。

## 文件结构

- `index.html`：页面入口，包含 Canvas 和基础 UI
- `styles.css`：页面样式，让画布居中并保持像素风
- `src/main.js`：启动游戏，创建 Canvas，运行游戏循环
- `src/input.js`：统一处理键盘和手柄输入
- `src/entities.js`：玩家、小怪、Boss 和攻击判定
- `src/levels.js`：平台、出生点、敌人位置和 Boss 区域
- `src/render.js`：绘制角色、敌人、平台、血条和提示文字

## 运行流程

1. 浏览器打开 `index.html`
2. `main.js` 初始化 Canvas、输入和关卡
3. 游戏循环每一帧读取统一输入状态
4. 更新玩家、敌人、Boss、攻击、碰撞和游戏状态
5. `render.js` 绘制当前画面
6. 玩家死亡时显示失败提示
7. Boss 被击败时显示胜利提示
8. 按重开键重新初始化关卡

## 开发与验收步骤

每一步都要能在浏览器运行并观察结果：

1. 建立页面和 Canvas，看到空游戏画面
2. 绘制玩家，让玩家站在地面上
3. 加键盘输入，让玩家左右移动
4. 加重力和跳跃，让玩家能起跳并落地
5. 加平台碰撞，让玩家能跳上平台
6. 加手柄输入，让手柄和键盘控制同一套动作
7. 加剑攻击判定，能看到攻击范围
8. 加小怪巡逻、受伤和死亡
9. 加玩家受伤、无敌时间和失败状态
10. 加 Boss 冲撞、血量和胜利状态
11. 加像素风绘制、血条、提示文字和重开

## 成功标准

- 游戏能在浏览器中打开并运行
- 键盘可以完成移动、跳跃、攻击和重开
- 支持标准浏览器手柄输入
- 玩家能跳上平台
- 玩家能用剑击败小怪
- 玩家能和 Boss 战斗
- Boss 被击败后显示胜利
- 玩家生命归零后显示失败
- 第一版代码结构清楚，适合继续逐步讲解和扩展
